# 要件定義

## 目的
- 営業用の仮サイトに「Usage（このサイトの使い方）」ページを追加する（顧客向けではない）。

## ルート／ナビ
- ルート: `/usage`。
- ナビ表示名: 「このサイトの使い方」。
- ナビに表示する（サンプル用途）。

## ページの役割／禁止事項
- サービス説明ページではない。
- 売り文句・プロモーション文言の追加はしない。
- 技術説明の深掘りはしない。
- FAQ／Notice内容の再掲はしない。
- 完成版・公開前提の文言は使わない。
- 「営業でどう使うか」を手順として整理するページにする。

## ページ上部
- 見出し: 「Usage」。
- サブ見出し: 「このサイトの使い方（営業・説明用）」。
- リード文（仮テキスト）:
  - 「このページは、営業・社内説明用の導線ガイドとして作成した仮ページです。」
  - 「説明時は、相手の理解度や目的に応じて、見るページを取捨選択してください。」

## Notice帯
- ページ最上部に以下のNotice帯を表示:
  - 「※本ページは営業・説明用途を想定した構成検討用の仮サイトです。」

## UI / レイアウト
- 全体は「説明資料」寄りのデザイン。
- 派手な装飾は不要、セクション区切りを明確に。
- 図解風のステップ表示（アイコン可）。
- 余白多め、読みやすさ最優先。
- ヘッダー／フッターは既存レイアウトを使用。

## トーン
- 社内資料＋営業資料の中間。
- 断定しすぎないが、曖昧にもしない。

## 受け入れ条件
- `/usage` がヘッダー／フッターとNotice帯を含めて表示される。
- ナビに「このサイトの使い方」が表示され、`/usage` に遷移できる。
- ページ上部に指定の見出し・サブ見出し・リード文がある。
- 禁止事項を避けた上で、営業向けの手順整理として構成されている。
- UI/トーンの制約に沿っている。
