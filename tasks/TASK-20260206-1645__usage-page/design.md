# 詳細設計

## ページ構成
1. Notice帯（ページ最上部、固定文言）
2. ヒーロー/イントロ
   - 見出し: Usage
   - サブ見出し: このサイトの使い方（営業・説明用）
   - リード文（2文）
3. セクションA: 「このページの目的」
   - 営業・社内説明のための導線ガイドであること
   - サービス説明ではないことを明記
4. セクションB: 「説明の進め方（ステップ）」
   - 図解風のステップ表示（3〜5ステップ）
   - 各ステップは短い見出し＋1〜2行説明
5. セクションC: 「状況別の見せ方の目安」
   - 相手の理解度/目的に応じた取捨選択のガイド
   - 具体例は軽め（売り文句は入れない）
6. セクションD: 「当日のチェックポイント」
   - 営業・引き継ぎ時の確認項目を簡潔に列挙

## コンテンツ方針
- 断定しすぎず、曖昧にもならない文体。
- プロモーション/売り文句は避ける。
- 技術詳細やFAQに踏み込まない。
- 完成版前提の表現は使わない。

## UI / レイアウト
- 既存ヘッダー/フッターを流用。
- 背景は控えめ、余白多めで読みやすさ優先。
- セクションはカード/枠線/薄い背景で明確に区切る。
- ステップは横並び or 縦並びの図解風（番号＋アイコン可）。
- 視線誘導はシンプルに（装飾よりも情報の整理を優先）。

## 追加コンポーネント
- Notice帯: 既存スタイルがなければ簡易な帯を新規作成。
- ステップ表示: 既存のセクション/カードを流用し、番号と短文で構成。

## 文言（固定）
- Notice帯: 「※本ページは営業・説明用途を想定した構成検討用の仮サイトです。」
- 見出し: 「Usage」
- サブ見出し: 「このサイトの使い方（営業・説明用）」
- リード文:
  - 「このページは、営業・社内説明用の導線ガイドとして作成した仮ページです。」
  - 「説明時は、相手の理解度や目的に応じて、見るページを取捨選択してください。」
